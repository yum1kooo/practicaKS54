# Практика 1 день

1. Main

Назначение: Точка входа программы. Управляет пользовательским интерфейсом через консоль, загружает и сохраняет базу данных задач.

Основные действия:
Загружает сохранённую базу данных из файла database.dat


Предоставляет консольное меню с опциями:
Создать задачу
Показать все задачи
Изменить имя задачи
Изменить описание задачи
Изменить статус задачи
Удалить задачу
Выйти и сохранить базу
Сохраняет базу в database.dat при выходе

Взаимодействие с другими классами: Использует Person для управления задачами и DataBase для хранения задач.

2. Person
Назначение: Представляет пользователя системы и содержит методы для работы с его задачами.

Поля:
name — имя пользователя.
scanner — для ввода с консоли (transient, не сериализуется).
db — объект класса DataBase.

Основные методы
createNewTask() — создаёт новую задачу, спрашивает у пользователя имя, описание, статус и ID, добавляет в базу.
removeTask() — удаляет задачу по ID с подтверждением.
changeTaskName(), changeTaskDescription(), changeTaskStatus() — изменяют соответствующие поля задачи по ID.
getName() и setName(String) — стандартные геттер и сеттер имени.

Обработка через InputMismatchException от некоректного ввода


3. DataBase
Назначение: Хранит задачи в памяти.

Поля:
taskDB — LinkedHashMap<Integer, Task> для хранения задач с их ID.

Основные методы:
getAllTask() — выводит все задачи с их ID и именем (ключ id, значение Task)

addTask(int id, Task task) — добавляет задачу в базу (больше как тестовый метод или для пременения в методах для пользователся есть иной)

searchTaskById(int id) — ищет задачу по ID (для методов)

removeTaskById(int id) — удаляет задачу по ID (для методов)

toString() — просто переопределение


4. Task

Назначение: Для хранения полей задачи

Поля:
name — название задачи.
description — описание задачи.
status — статус задачи.
scanner — transient Scanner (тк сканер не сереализуеться)

Методы:

Сеттеры и геттеры для всех полей (name, description, status).
toString() — просто переопределение


Взаимодействие всех компонентов
Main управляет программой и взаимодействует с Person.
Person управляет задачами через DataBase.
DataBase хранит объекты Task и предоставляет методы поиска, добавления и удаления.
